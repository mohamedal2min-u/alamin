# ميثاق مشروع — نظام إدارة مداجن دجاج اللحم
**Dajajati Broiler Farm Management System**
**الإصدار:** 1.0 | **التاريخ:** 2026-04-11

---

## 1. تعريف المشروع

**دجاجاتي** منصة SaaS عربية متعددة المستأجرين لإدارة وتشغيل ومحاسبة مداجن دجاج اللحم (Broiler).  
تتيح لصاحب المزرعة متابعة كل دورة إنتاج من يوم الإدخال حتى يوم البيع، مع رؤية مالية كاملة.

---

## 2. المشكلة التي يحلّها المشروع

| الوضع الحالي | الهدف المنشود |
|---|---|
| تسجيل يدوي بالورق أو Excel | منصة رقمية متكاملة |
| لا رؤية فورية لنفوق القطيع | تنبيهات فورية عند تجاوز الحدود |
| لا مقارنة بين الدورات | تقارير تاريخية ومقارنة مالية |
| صعوبة حساب تكلفة الكيلو | حساب تلقائي لـ FCR وتكلفة الإنتاج |
| لا متابعة للعمال والعنابر | إدارة تشغيلية يومية منظّمة |

---

## 3. الأدوار والمستخدمون

| الدور (كود) | الاسم | الصلاحيات الرئيسية |
|---|---|---|
| `super_admin` | المشرف العام | إدارة المنصة كاملة، الخطط، المزارع، التقارير العامة |
| `farm_admin` | مدير المزرعة | كل شيء داخل مزرعته فقط |
| `farm_manager` | مشرف المزرعة | إدخال البيانات، لا حذف، لا إعدادات |
| `worker` | عامل | رؤية مهامه، تسجيل الدخول المحدود |

---

## 4. نطاق المشروع (Scope)

### ✅ داخل النطاق
- إدارة القطعان (Flock) — دورة حياة كاملة من يوم 0 إلى الحصاد
- السجلات اليومية — نفوق، وزن، استهلاك علف، درجة حرارة
- إدارة المخزون — أعلاف، أدوية، مطاعيم، مستلزمات
- المبيعات والحصاد — فواتير، حصاد جزئي وكلي
- المصاريف — تشغيلية، عمالة، صيانة
- إدارة العمال — توظيف، مهام، حضور
- تقارير مالية وتشغيلية — FCR، تكلفة الكيلو، ربح/خسارة الدورة
- لوحة تحكم Super Admin — إحصاءات المنصة

### ❌ خارج النطاق (v1)
- تطبيق Flutter — مرحلة v2
- تكامل مع أنظمة ERP خارجية
- نظام محاسبي كامل (فواتير ضريبية رسمية)
- بحوث التغذية أو البروتوكولات البيطرية

## 5. القرارات التقنية الثابتة

| القرار | الاختيار | السبب |
|---|---|---|
| Backend API | **Laravel 11** | نضج، Eloquent، Sanctum مدمج |
| Frontend | **Next.js 15 (App Router)** | SSR، SEO، أداء ممتاز |
| Mobile | **Flutter 3** | تطبيق واحد لـ iOS وAndroid |
| قاعدة البيانات | **PostgreSQL 16** | Row Level Security، JSONB، أداء |
| المصادقة | **Laravel Sanctum** | Token-based API، SPA support |
| الصلاحيات | **Spatie Laravel Permission** | roles & permissions مرن |
| Multi-tenancy | **farm_id على كل جدول** | Row-level isolation |
| اللغة | **العربية فقط — RTL** | الجمهور المستهدف |
| العملة | **USD (دولار أمريكي)** | معيار دولي موحّد |

---

## 6. حدود العزل والتشغيل

```
حد العزل الأساسي  →  farm_id
                       كل query تحمل farm_id إلزامياً

مركز التشغيل الأساسي → flock (القطيع)
                       كل العمليات اليومية مرتبطة بـ flock نشط
```

---

## 7. قيود المشروع الصارمة

1. لا يبدأ أي كود API قبل اعتماد `database_schema.md` كاملاً
2. لا يُنشأ أي ملف توثيق غير الستة المحددة في `docs/`
3. كل تغيير معماري يُسجَّل في `change-log.md` قبل التنفيذ
4. كل جدول يحمل `farm_id` بدون استثناء (عدا جداول Lookup)
